### Шаг 2.10: Привяжите ваш домен

1. В настройках сайта найдите раздел **"Домены"** или **"Domains"**

3. Введите ваш домен, например: **`ваш-домен.ru`**
4. TimeWeb покажет инструкции по настройке DNS
5. Настройте DNS у вашего регистратора домена:

   **Вариант А (CNAME - проще):**
   - Создайте **CNAME запись**: `@` → указанный TimeWeb адрес

   **Вариант Б (A запись):**
   - Создайте **A запись**: `@` → IP адрес TimeWeb (узнайте в панели)
   - Или `www` → IP адрес TimeWeb

## 🆘 Если что-то не работает

### Проблема: Сборка не запускается или ошибка "Could not resolve entry module index.html"

**Решение:**
1. Проверьте логи в панели TimeWeb (раздел "Логи" или "Logs")
2. Убедитесь, что команда сборки правильная:
   - Попробуйте: `cd Helper2 && npm install && npm run build`
   - Или: `bash -c "cd Helper2 && npm install && npm run build"`
   - Или: `npm install && npm run build` (если есть поле "Рабочая директория" = `Helper2`)
3. Проверьте, что папка с результатом правильная:
   - Попробуйте: `Helper2/dist`
   - Или: `dist` (если команда выполняется из `Helper2`)
4. Проверьте версию Node.js (должна быть 18 или 20)
5. Убедитесь, что файл `Helper2/index.html` существует в репозитории
6. Проверьте, что файл `Helper2/vite.config.ts` существует и не содержит ошибок

### Проблема: Ошибка сборки "Cannot find module"

**Решение:**
1. Убедитесь, что все зависимости указаны в `package.json`
2. Попробуйте очистить кэш и пересобрать
3. Проверьте логи сборки на наличие конкретных ошибок

### Проблема: Яндекс Диск не работает (ошибка загрузки файлов)

**Решение:**
1. Проверьте переменную `VITE_BACKEND_API_URL`:
   - Должен быть URL бэкенда БЕЗ `/api/yandex-disk` в конце
   - Например: `https://otvertka-backend.timeweb.ru` ✅
   - НЕ: `https://otvertka-backend.timeweb.ru/api/yandex-disk` ❌
2. Убедитесь, что бэкенд работает (откройте его URL в браузере)
3. Проверьте CORS настройки в бэкенде (должен разрешать ваш домен)
4. Откройте консоль браузера (F12) и посмотрите ошибки

### Проблема: Домен не открывается

**Решение:**
1. Проверьте DNS настройки у регистратора домена
2. Убедитесь, что прошло достаточно времени (DNS может применяться до 24 часов)
3. Проверьте, что домен правильно привязан в панели TimeWeb
4. Попробуйте открыть временный URL TimeWeb (должен работать сразу)

### Проблема: Страница белая или показывает ошибку

**Решение:**
1. Откройте консоль браузера (F12 → Console)
2. Посмотрите ошибки (они будут красным цветом)
3. Проверьте переменные окружения в панели TimeWeb
4. Убедитесь, что все переменные начинаются с `VITE_`
5. Пересоберите проект после изменения переменных
