# Тестирование системы уведомлений

## ✅ Что уже сделано

1. SQL скрипт выполнен - таблица `notifications` создана в Supabase
2. Компонент `NotificationSystem` подключен в `App.tsx`
3. Уведомления интегрированы в:
   - Создание задач (`ScheduleViewWithAPI.tsx`)
   - Создание отчётов (`ReportsViewWithAPI.tsx`)
   - Создание дефектов (`CreateDefectForm.tsx`, `DefectsView.tsx`)
   - Изменение прогресса (`ProgressTable.tsx`)

## 🧪 Как протестировать

### Тест 1: Уведомление о задаче (реальное время)

1. **Откройте приложение в двух браузерах/вкладках:**
   - Браузер 1: Войдите как пользователь А (например, технадзор)
   - Браузер 2: Войдите как пользователь Б (например, рабочий)

2. **В браузере 1 (пользователь А):**
   - Перейдите в "График работ" (Schedule)
   - Создайте новую задачу
   - Назначьте её пользователю Б (выберите его в поле "Исполнитель")

3. **В браузере 2 (пользователь Б):**
   - **БЕЗ ОБНОВЛЕНИЯ СТРАНИЦЫ** должно появиться уведомление в правом верхнем углу
   - Уведомление будет с иконкой задачи и текстом: "Новая задача назначена"

### Тест 2: Уведомление об отчёте (реальное время)

1. **В браузере 1 (пользователь А):**
   - Перейдите в "Отчёты"
   - Нажмите "Создать отчёт"
   - Заполните форму:
     - Выберите проект
     - Выберите получателя (пользователь Б)
     - Выберите тип отчёта
     - Заполните описание
   - Нажмите "Создать отчёт"

2. **В браузере 2 (пользователь Б):**
   - **БЕЗ ОБНОВЛЕНИЯ СТРАНИЦЫ** должно появиться уведомление
   - Уведомление будет с иконкой отчёта и текстом: "Новый отчёт получен"

### Тест 3: Временное уведомление (дефект)

1. **В любом браузере:**
   - Создайте дефект (через планы или форму)
   - После создания должно появиться временное уведомление
   - Уведомление исчезнет автоматически через 3 секунды

### Тест 4: Временное уведомление (прогресс)

1. **В любом браузере:**
   - Перейдите в "Таблица прогресса"
   - Измените процент выполнения любой работы
   - После сохранения должно появиться временное уведомление
   - Уведомление исчезнет автоматически через 3 секунды

## 🔍 Проверка в консоли браузера

Откройте консоль разработчика (F12) и проверьте:

1. **При загрузке страницы:**
   - Должны быть сообщения о загрузке уведомлений
   - Должно быть сообщение о подписке на Realtime канал

2. **При создании уведомления:**
   - Должно быть сообщение: `✅ Уведомление создано: {...}`

3. **При получении уведомления в реальном времени:**
   - Должно быть сообщение: `🔔 Новое уведомление получено в реальном времени: {...}`

## ⚠️ Возможные проблемы

### Уведомления не появляются

1. **Проверьте консоль браузера:**
   - Есть ли ошибки подключения к Supabase?
   - Есть ли ошибки RLS (Row Level Security)?

2. **Проверьте Realtime:**
   - В Supabase Dashboard → Database → Replication
   - Убедитесь, что таблица `notifications` включена в Realtime

3. **Проверьте RLS политики:**
   - В Supabase Dashboard → Authentication → Policies
   - Убедитесь, что политики для таблицы `notifications` активны

### Уведомления появляются только после обновления страницы

- Это означает, что Realtime не работает
- Проверьте, что в SQL скрипте была выполнена строка:
  ```sql
  ALTER PUBLICATION supabase_realtime ADD TABLE notifications;
  ```

## 📝 Дополнительная информация

- **Постоянные уведомления** (persistent: true) - требуют закрытия вручную (кнопка X)
- **Временные уведомления** (persistent: false) - исчезают автоматически через 3 секунды
- Уведомления автоматически помечаются как прочитанные при наведении мыши
- Уведомления сохраняются в базе данных и загружаются при следующем входе
