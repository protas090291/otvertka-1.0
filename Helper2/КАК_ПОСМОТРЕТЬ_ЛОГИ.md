# Как посмотреть логи Edge Functions в Supabase

## Способ 1: Через Supabase Dashboard (веб-интерфейс)

1. **Откройте Supabase Dashboard:**
   - Перейдите по ссылке: https://supabase.com/dashboard/project/yytqmdanfcwfqfqruvta

2. **Перейдите в раздел Edge Functions:**
   - В левом меню найдите раздел **"Edge Functions"** (или **"Functions"**)
   - Нажмите на него

3. **Выберите функцию `yandex-disk`:**
   - В списке функций найдите `yandex-disk`
   - Нажмите на неё

4. **Откройте вкладку "Logs":**
   - В открывшемся окне перейдите на вкладку **"Logs"** (или **"Логи"**)
   - Там будут видны все логи выполнения функции

5. **Фильтрация логов:**
   - Можно фильтровать по времени (последний час, день, неделя)
   - Можно искать по ключевым словам (например, "REQUEST PARAMS", "RESPONSE DATA")

## Способ 2: Через Supabase CLI (терминал)

Если у вас установлен Supabase CLI, можно посмотреть логи через терминал:

```bash
# Перейти в директорию проекта
cd "/Users/zaharprotas/Desktop/git test hub/otvertka-demo/Helper2"

# Посмотреть логи функции yandex-disk
npx supabase functions logs yandex-disk

# Посмотреть логи в реальном времени (следить за новыми логами)
npx supabase functions logs yandex-disk --follow

# Посмотреть логи за последний час
npx supabase functions logs yandex-disk --since 1h

# Посмотреть логи за последний день
npx supabase functions logs yandex-disk --since 24h
```

## Что искать в логах

При отладке проблемы с Яндекс Диском ищите следующие записи:

1. **`=== REQUEST PARAMS ===`** - показывает, какой путь был передан в функцию
2. **`folder_path param (raw)`** - исходный путь от frontend
3. **`folder_path param (normalized)`** - нормализованный путь
4. **`folderPath (final)`** - финальный путь, который будет использован
5. **`Normalized path for API`** - путь, который отправляется в Яндекс API
6. **`Final Yandex URL`** - полный URL запроса к Яндекс API
7. **`Response status`** - статус ответа от Яндекс API (200 = успех, 404 = не найдено)
8. **`=== RESPONSE DATA ===`** - данные, полученные от Яндекс API
9. **`Total items received`** - количество файлов/папок в ответе
10. **`Yandex API error`** - ошибки от Яндекс API (если есть)

## Пример логов

Вот как выглядят логи при успешном запросе:

```
=== REQUEST PARAMS ===
action: files
folder_path param (raw): disk:/T0201 Luxury
folder_path param (normalized): /T0201 Luxury
folderPath (final): /T0201 Luxury

=== YANDEX DISK EDGE FUNCTION DEBUG ===
Action: files
Folder path: /T0201 Luxury
Has publicKey: true

Normalized path for API: T0201 Luxury
Final Yandex URL: https://cloud-api.yandex.net/v1/disk/public/resources?public_key=...

Response status: 200
=== RESPONSE DATA ===
Total items received: 15
```

При ошибке 404 будет:

```
Response status: 404
Yandex API error 404: {"error":"DiskNotFoundError","description":"Resource not found."}
```
